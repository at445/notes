### `StringRef` vs `StringLiteral`

两者都是不拥有内存的字符串视图（只存指针和长度），但在约束和使用场景上有严格区别。

| 特性           | `llvm::StringRef`                                           | `llvm::StringLiteral`                           |
| -------------- | ----------------------------------------------------------- | ----------------------------------------------- |
| **接受的数据** | **动态和静态均可**（`std::string`, `const char*`, `"..."`） | **仅限硬编码字面量**（`"..."`）                 |
| **长度计算**   | 运行时计算（如果是 `char*` 需调 `strlen`）                  | **编译期零开销**（通过模板推导数组大小）        |
| **内存安全性** | **需开发者保证**（可能指向堆/栈内存，易成悬垂指针）         | **绝对安全**（强制指向只读数据段 `.rodata`）    |
| **实现原理**   | 隐式转换为 `const char*`                                    | 模板严格匹配定长字符数组引用 `const char(&)[N]` |
| **典型场景**   | 解析器子串、接收用户输入（如命令行路径）、配合 Arena 使用   | 编译器内置类型名、固定错误信息、Opcode 名字     |

总结代码：

```c++
const char* dyn_str = argv[1];
llvm::StringRef r1(dyn_str);       // ✅ OK
llvm::StringLiteral l1(dyn_str);   // ❌ 编译报错：无法推导数组大小
llvm::StringRef r2("main.toy");    // ✅ OK (有 strlen 开销)
llvm::StringLiteral l2("main.toy");// ✅ OK (完美匹配，零开销，最安全)
```



### C++ `const` 关键字的位置

- #### 1. 前置 `const` (修饰返回值)

  - **作用对象**：函数的返回值。
  - **核心规则**：对于**引用 (`&`)** 或 **指针 (`\*`)** 语义的返回值，前置 `const` 用于**保护类的内部资产不被外部篡改**。
  - **调用者视角**：调用者拿到的是数据的“只读视图”。你可以借去看，但编译器限制你不能修改原版数据。
    *(注：对于值语义返回值，前置 `const` 意义不大，因为传递的是全新的拷贝。)*

  **【代码示例】**

  ```c++
  class Employee {
      std::string name = "Jasson";
  public:
      // 前置 const：返回只读引用，外部无法修改 name
      const std::string& getName() { return name; } 
      
      // 无前置 const：返回裸引用，极度危险，外部可以直接修改内部变量
      std::string& getNameDanger() { return name; } 
  };
  
  void test(Employee& emp) {
      const std::string& safe_name = emp.getName();
      // safe_name = "Hacker"; // ❌ 编译报错：禁止篡改
      
      emp.getNameDanger() = "Hacker"; // ✅ 编译通过：对象的内部状态在外部被改写
  }
  ```

  

  ------

  #### 2. 后置 `const` (修饰成员函数)

  - **作用对象**：类的成员函数（本质上修饰的是隐藏的 `this` 指针，使其变为 `const T* this`）。
  - **核心规则**：向编译器签署保证书：**本方法执行期间，绝对不修改类的任何成员变量**。
  - **调用者视角**：这是专门为了**接纳只读级别的调用者**而设立的。只要加了后置 `const`，无论调用者是普通对象（`T*`）还是只读对象（`const T*`），都可以合法调用该方法；若不加，只读对象将被直接剥夺调用权限。

  **【代码示例】**

  ```c++
  class ExprAST {
      int val = 42;
  public:
      // 无后置 const：只允许拥有“读写权限”的普通对象调用
      int getValueMutable() { return val; } 
  
      // 有后置 const：承诺不改状态，允许“只读权限”的对象调用
      int getValueReadOnly() const { return val; } 
  };
  
  void test(const ExprAST* read_only_node) {
      // ❌ 编译报错：read_only_node 只有读权限，但 getValueMutable 未承诺不修改数据，禁止放行！
      // int a = read_only_node->getValueMutable(); 
  
      // ✅ 编译通过：getValueReadOnly 已承诺期间不修改数据，安全放行！
      int b = read_only_node->getValueReadOnly(); 
  }
  
  void test( ExprAST* node) {
      // ✅ 编译通过：因为 node 有读写权限，门当户对。
      int a = node->getValueMutable(); 
  
      // ✅ 编译通过：因为 node 有读写权限，向下兼容（权限降级），安全放行。
      int b = node->getValueReadOnly(); 
  }
  ```

  

  ### 💡 核心记忆口诀

  - **前置 `const`** = 防外贼（防止外部调用者修改我返回的内部数据）。
  - **后置 `const`** = 款待贵客（向编译器自证清白，从而允许 `const` 类型的只读对象调用我的方法）。
